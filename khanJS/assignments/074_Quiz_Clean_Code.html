Quiz: Clean Code

Check if you have a good idea of what makes JavaScript code readable and understandable.

Question 1:

Which of these code snippets is the most readable, according to the JavaScript coding style conventions that we recommend here?

// addUp
		
// Answer 1
var addUp = function(a, b) {
  var c = a + b;
  return c;
};

// Answer 2
var addUp = function(a, b) {
var c = a + b;
return c;
};

// Answer 3
var addUp = function(a, b) { var c = a + b; return c; };

According to most JavaScript coding style guidelines, you should put the body of your function definition on a different line from the header of the function definition. Also, you should indent the body one level, so that you can easily see that the code is inside the function definition. Therefore, Answer 1 is the most readable.

According to most JavaScript coding style guidelines, you should put the body of your loop on a different line from the header of the for loop. Also, you should indent the body of the for loop one level, so that you can easily see that the code is inside the for loop. Therefore, Answer 1 is the most readable.

Question 3:

Which of these code snippets is the most readable, according to the JavaScript coding style conventions that we recommend here?

// Answer 1
if (x > 2) {
println(x);
}

// Answer 2
if (x > 2) {println(x);}

// Answer 3
if (x > 2) {
  println(x);
}

According to most JavaScript coding style guidelines, you should put the body of your if statement on a different line from the condition of the if statement. Also, you should indent the body one level, so that you can easily see that the code is inside the if statement. Therefore, Answer 3 is the most readable.

Question 4:

Which of these code snippets is the most readable, according to the JavaScript coding style conventions that we recommend here?

// Answer 1
ellipse(x, y, 15, 5);

// Answer 2
ellipse(x,y,15,5);

// Answer 3
ellipse (x, y, 15, 5);

According to most JavaScript coding style guidelines, you should put spaces after the parameters in your function calls, and no spaces between the function name and the parentheses. Therefore, Answer 1 is the most readable.

Question 5:

Which of these code snippets is the most readable, according to the JavaScript coding style conventions that we recommend here?

// Answer 1
var x = 5*i;

// Answer 2
var x = 5 * i;

// Answer 3
var x=5 * i;

According to most JavaScript coding style guidelines, you should put spaces before and after the assignment operator (=) and a space before and after mathematical operators (like *). Therefore, Answer 2 is the most readable.

Question 6:

Which of these code snippets is the most readable, according to the JavaScript coding style conventions that we recommend here? 

// Answer 1
if (x>5 && y<10) {
  rect(x, y, 10, 10);
}

// Answer 2
if (x > 5 && y < 10) {
  rect(x, y, 10, 10);
}

// Answer 3

  rect(x, y, 10, 10);

According to most JavaScript coding style guidelines, you should put spaces before and after logical operators (like &&) and a space before and after comparison operators (like >, <). Therefore, Answer 2 is the most readable.

Question 8:

Let's say you wrote a program that animated a Superman flying across the screen. Which would be the best comment to put at the top? 

// Answer 1
/* This program is so cool, you should vote it up.
*/

// Answer 2
/* superman!!
*/

// Answer 3
/* 
This program animates Superman flying across the screen.
The speed is controlled by variables at the top.
Written in 2015 by @Batman.
*/

It's a great idea to put a multi-line comment at the top of your programs that describes what it does, gives the viewer any relevant instructions, and provides information on the author. The best comment of the ones provided is Answer 3.

Question 9:

// Answer 1

// Answer 2

// Answer 3

It's a good idea to comment lines of your program that might be confusing to others, especially if you're sharing with an audience of beginner programmers.
